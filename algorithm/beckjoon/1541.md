### 1541_잃어버린 괄호

> ## 문제
>
> 세준이는 양수와 +, -, 그리고 괄호를 가지고 식을 만들었다. 그리고 나서 세준이는 괄호를 모두 지웠다.
>
> 그리고 나서 세준이는 괄호를 적절히 쳐서 이 식의 값을 최소로 만들려고 한다.
>
> 괄호를 적절히 쳐서 이 식의 값을 최소로 만드는 프로그램을 작성하시오.
>
> ## 입력
>
> 첫째 줄에 식이 주어진다. 식은 ‘0’~‘9’, ‘+’, 그리고 ‘-’만으로 이루어져 있고, 가장 처음과 마지막 문자는 숫자이다. 그리고 연속해서 두 개 이상의 연산자가 나타나지 않고, 5자리보다 많이 연속되는 숫자는 없다. 수는 0으로 시작할 수 있다. 입력으로 주어지는 식의 길이는 50보다 작거나 같다.
>
> ## 출력
>
> 첫째 줄에 정답을 출력한다.



- 이게 왜 그리디라는건지 잘 모르겠다... 애초에 -전후로 미리 계산해서 마지막에 빼면 된다는 아이디어 생각해내는 것도 힘들다. 
- -면 다음 -나오기 전까지 그 안에 +있으면 다 계산하고 +면 어차피 +할 수 밖에 없으니까 계산한다. 

```python
data = input().split('-')

result = []
for i in data:
    temp = i.split('+')
    # 길이가 1인 애들은 앞에 -가 붙어있던 숫자들이다.
    if len(temp) == 1:
        result.append(int(temp[0]))
        continue
    # +로 이루어진 덩어리다.
    temp = list(map(int, temp))
    # +로 이루어진 덩어리 전부 +
    result.append(sum(temp))
# 계산식 처음은 무조건 숫자로 시작한다.
ans = result[0]
for i in range(1, len(result)):
    # +였던 애들의 합, -붙어있던 애들 다 빼준다.
    ans -= result[i]

print(ans)
```
